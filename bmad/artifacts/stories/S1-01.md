# Story S1-01 – Add `mongo-test` sidecar to `docker-compose.yml`

**Epic:** Data Safety & Test Infrastructure
**Sprint:** 01
**Points:** 1
**Priority:** Must
**Traces to:** FR-01 (PRD), ADR-01 (Architecture)

---

## User Story

As a **developer**, I want a dedicated MongoDB container for tests inside Docker Compose,
so that PHPUnit never connects to the production database.

---

## Context

Production MongoDB runs on the host at `host.docker.internal:27017` and holds live data.
All test code must connect to an isolated `mongo-test` sidecar (mongo:7, port 27018 on host,
27017 internally). The `app` container must expose `MONGO_TEST_HOST` and `MONGO_ENV` env vars
so PHPUnit bootstrap can connect to the right instance.

Note: `docker-compose.yml` has an existing bug — a comment is concatenated inline on the
`DOCKER_ENV` env var line. Fix it in this story.

---

**After (target):**
```yaml
services:
  app:
    build: .
    ports:
      - "8080:80"
    volumes:
      - ./idae:/var/www/html/idae
      - ./logs:/var/log/apache2
    depends_on:
      - mongo-test
    environment:
      - APACHE_RUN_USER=www-data
      - APACHE_RUN_GROUP=www-data
      - CONF_INC=/var/www/html/idae/web/conf.inc.php
      - DOCKER_ENV=1
      - MONGO_HOST=host.docker.internal
      - MONGO_HOST_PORT=27017
      - MDB_USER=${MDB_USER:-}
      - MDB_PASSWORD=${MDB_PASSWORD:-}
      - MDB_PREFIX=${MDB_PREFIX:-}
      - MONGO_ENV=${MONGO_ENV:-dev}
      - MONGO_TEST_HOST=mongo-test
      - MONGO_TEST_PORT=27017
      - MONGO_TEST_DSN=mongodb://mongo-test:27017
      - DEBUG_DB=1
      - DEBUG_SESSION=1
      - SOCKET_PORT=3005
    container_name: idae-legacy
    healthcheck:
      test: ["CMD", "curl", "-f", "-m", "3", "http://localhost/idae/web/check_mongo.php"]
      interval: 30s
      timeout: 5s
      retries: 3
      start_period: 40s
    restart: unless-stopped

  # Isolated MongoDB instance for PHPUnit tests.
  # Never receives production data. No auth required (test-only).
  mongo-test:
    image: mongo:7
    container_name: idae-mongo-test
    ports:
      - "27018:27017"
    tmpfs:
      - /data/db   # in-memory storage: fast + auto-wiped on container stop
    restart: "no"  # test DB should not auto-restart in production
```

---

## Technical Notes

- `tmpfs: /data/db` — stores test data in RAM. Faster than disk, automatically wiped when the container stops. Perfect for test isolation.
- `MONGO_ENV=${MONGO_ENV:-dev}` — defaults to `dev`. Override with `MONGO_ENV=test` when running PHPUnit (done in phpunit.xml bootstrap in S1-04).
- `depends_on: mongo-test` — ensures `mongo-test` starts before `app`. Does not wait for MongoDB to be ready; PHPUnit bootstrap handles retry logic (S1-03).
- No MongoDB auth on sidecar — test environment only. The guard in `ClassMongoDb` (S1-02) enforces this can't be confused with prod.
- **Do not add** `mongo-test` to the healthcheck endpoint — it's test-only and should not affect app container health status.
