# Story S3-06 – json_data.php regression tests

**Epic:** ClassApp CRUD Migration
**Sprint:** 03
**Points:** 3
**Priority:** High
**Traces to:** services/json_data.php, S3-03, S3-05

---

## User Story

As a tester, implement regression/integration tests that exercise `services/json_data.php` endpoints to ensure the modernized ClassApp CRUD and FK helpers do not break legacy API JSON shapes consumed by the frontend.

---

## Context

- The frontend depends on exact JSON shapes for grids and forms; regression tests must assert schema compatibility.
- Tests should simulate authenticated agent context where necessary.
- Use a test MongoDB instance and Dockerized PHP stack when running integration scenarios.

---

## Acceptance Criteria

- Regression tests invoke `services/json_data.php` for list, create, update, delete scenarios and assert response status and JSON shapes.
- Any differences between legacy and modernized outputs are documented and an adapter added where needed.
- Integration tests pass in CI/local Docker when the test stack is started.

---

## Execution Plan

1. Add integration test script `idae/web/tests/Integration/JsonDataRegressionTest.php`.
2. Seed test database with minimal fixtures.
3. Use HTTP client (Guzzle or curl) to call `services/json_data.php` endpoints and assert JSON shape.
4. Run integration tests in Docker and fix regressions.
5. Commit tests and update `bmad/status.yaml` and dashboard.

---

## Notes

- Keep test data minimal and reset DB between runs.
- Mark slow integration tests so CI can opt-in.
