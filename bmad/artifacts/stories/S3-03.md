# Story S3-03 – Refactor ClassApp create_update() and FK/aggregation helpers

**Epic:** ClassApp CRUD Migration  
**Sprint:** 03  
**Points:** 3  
**Priority:** Must  
**Traces to:** PHASE2_STRATEGY.md, MONGOCOMPAT.md

---

## User Story

As a back-end developer, refactor `ClassApp::create_update()` and related FK/aggregation helpers so upsert, FK lookups and aggregation helpers use the modern MongoDB driver and MongoCompat conversions, preserving legacy semantics and return shapes.

---

## Context

- Builds on S3-01/S3-02 changes (query/findOne and CRUD write methods).  
- FK helpers (`get_grille_fk`, `get_reverse_grille_fk`) and aggregation-friendly helpers are widely used by templates and services and must remain backward-compatible.  
- Tests must run under `MONGO_ENV=test` against `mongo-test` sidecar.

---

## Acceptance Criteria

- `create_update()` uses `updateOne(..., ['upsert' => true])` and returns logical ID as before.  
- FK helpers return the same array shape as legacy code (field names suffixed with table name).  
- All inputs pass through `MongoCompat::convertFilter()` and documents through `MongoCompat::convertDocument()` where applicable.  
- Unit tests added for `create_update()` upsert paths and FK helper behavior.  
- PHPStan reports no new issues for touched files.
