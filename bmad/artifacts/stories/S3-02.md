# Story S3-02 – Migrate ClassApp CRUD write methods to modern MongoDB driver

**Epic:** ClassApp CRUD Migration  
**Sprint:** 03  
**Points:** 3  
**Priority:** Must  
**Traces to:** Phase 2 Strategy (PHASE2_STRATEGY.md), MongoCompat (MONGOCOMPAT.md)

---

## User Story

As a **back-end developer**, I want `ClassApp::create_update()`, `insert()`, `update()`, and `remove()` migrated from the legacy MongoClient extension to the modern `MongoDB\Client` library,  
so that the core write operations use the current official MongoDB driver with proper type safety, error handling, and MongoCompat integration.

---

## Context

Following S3-01 (query/findOne ✅), the write-side CRUD methods still use legacy driver patterns. Currently:
- `getMongoClient()` returns `MongoDB\Client` (S2-05 completed)
- `create_update()`, `insert()`, `update()`, `remove()` need modernization
- Tests run against Docker test sidecar (`mongo-test` on port 27018)

This story:
1. **Refactors each method** to use modern MongoDB\Client driver semantics
2. **Adds unit tests** covering all write operations
3. **Validates MongoCompat** for all filter/document transformations
4. **Adds strict type hints** and English PHPDoc
5. **Ensures error handling** for connection and constraint violations

### Phase 2: Refactor create_update()

```php
/**
 * Create or update a document (upsert pattern).
 * 
 * Inserts a new document if the filter doesn't match any existing documents,
 * or updates the first matching document with the provided fields.
 * 
 * @param array<string, mixed> $vars Query filter (upsert key)
 * @param array<string, mixed> $fields Document fields to insert/update
 * @return int|string|null ID of inserted/updated document, or null on error
 * @throws \MongoDB\Driver\Exception\RuntimeException on connection error
 */
public function create_update(array $vars, array $fields = []): int|string|null {
    $vars = MongoCompat::convertFilter($vars);
    $fields = MongoCompat::convertDocument($fields);
    
    $result = $this->collection->updateOne(
        $vars,
        ['$set' => $fields],
        ['upsert' => true]
    );
    
    if ($result->getUpsertedId()) {
        return (int)$result->getUpsertedId();  // Newly inserted
    }
    
    // If updated, return the ID from filter
    return $vars['id' . ucfirst($this->table)] ?? null;
}
```

### Phase 3: Refactor insert()

```php
/**
 * Insert a new document.
 * 
 * @param array<string, mixed> $vars Document fields
 * @return int|string|null ID of inserted document, or null on error
 * @throws \MongoDB\Driver\Exception\RuntimeException on connection error
 */
public function insert(array $vars = []): int|string|null {
    $vars = MongoCompat::convertDocument($vars);
    
    $result = $this->collection->insertOne($vars);
    return $result->getInsertedId();
}
```

### Phase 4: Refactor update()

```php
/**
 * Update one or more documents.
 * 
 * @param array<string, mixed> $vars Query filter
 * @param array<string, mixed> $fields Fields to update
 * @param bool $upsert If true, insert new doc if no match (default: true)
 * @return int Count of updated documents
 * @throws \MongoDB\Driver\Exception\RuntimeException on connection error
 */
public function update(array $vars, array $fields = [], bool $upsert = true): int {
    $vars = MongoCompat::convertFilter($vars);
    $fields = MongoCompat::convertDocument($fields);
    
    $result = $this->collection->updateMany(
        $vars,
        ['$set' => $fields],
        ['upsert' => $upsert]
    );
    
    return $result->getModifiedCount();
}
```

### Phase 5: Refactor remove()

```php
/**
 * Delete one or more documents.
 * 
 * @param array<string, mixed> $vars Query filter
 * @return int Count of deleted documents
 * @throws \MongoDB\Driver\Exception\RuntimeException on connection error
 */
public function remove(array $vars = []): int {
    $vars = MongoCompat::convertFilter($vars);
    
    $result = $this->collection->deleteMany($vars);
    
    return $result->getDeletedCount();
}
```
