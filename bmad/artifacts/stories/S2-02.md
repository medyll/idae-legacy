# Story S2-02 – Replace `@{location_fragment}` with explicit `@use` in `main.scss` and entry points

**Epic:** SCSS Migration Complete  
**Sprint:** 02  
**Points:** 2  
**Priority:** Must  
**Traces to:** SCSS modernization, entry point consolidation

---

## User Story

As a **front-end developer**, I want all dynamic SCSS imports replaced with explicit `@use` and `@forward` directives,  
so that the stylesheet architecture is transparent, dependencies are explicit, and SCSS compilation is predictable and debuggable.

---

## Context

The SCSS pipeline currently uses `@use` and `@forward` properly in `main.scss`. This story ensures that:

1. **All entry points** in `appcss/scss/` use explicit `@use` imports (not dynamic `@{location_fragment}`)
2. **No dangling LESS import syntax** exists in SCSS files
3. **Proper `@use` namespacing** is applied (e.g., `@use "vars" as *` for global variables)
4. **Entry point CSS output** is correctly generated and validated

---

## Execution Plan

### Step 1: Audit Current Entry Points
List all SCSS entry point files (top-level and subdirectory imports):
- `main.scss` (primary)
- Check for secondary entry points or skin-specific imports

### Step 2: Verify @use/@forward Wiring
Ensure each entry point has:
- `@use "vars" as *;` (for global variables)
- `@use "app_fabric" as *;` (for mixin library)
- Proper `@use` for component-specific imports (table, global-tokens, appsite, datatable, etc.)
- NO `@import` statements (deprecated in SCSS)
- NO `@{location_fragment}` dynamic imports

Example correct pattern:
```scss
@use "vars" as *;
@use "app_fabric" as *;
@use "css/table";
@use "css/global-tokens";
@use "appsite";
@use "datatable";
```

### Step 3: Compile and Validate
1. Run `cd idae/web/app_node && npm run build:css`
2. Verify no compilation errors
3. Check `appcss/dist/main.css` exists and is valid CSS
4. Spot-check CSS output for completeness (e.g., presence of table, global-token rules)

### Step 4: Regression Testing
1. Boot the app: `docker-compose up`
2. Load the UI in browser (port 8080)
3. Check browser console for CSS-related errors
4. Verify page layout and styling are correct (no missing fonts, colors, spacing)

### Step 5: Document & Close
Update `main.scss` header with:
```scss
// Modified: 2026-03-02
// Changes: All @use/@forward directives confirmed; no dynamic imports.
```
